Passcode-Protected Quiz System

Google Sheets + Apps Script backend, static HTML frontend, on-page results dashboard.
quiz.html Code.gs

Two files make up this system: a static quiz.html page (hosted anywhere, e.g. GitHub Pages) and a Code.gs Apps Script backend bound to a private Google Sheet.

01Overview

Students open quiz.html, enter a passcode, fill in their name/ID, and answer questions. Submissions are graded and logged server-side into a Google Sheet. After the due date, the same passcode unlocks a results dashboard on the same page instead of the quiz form.

The answer key, passcode value, and due date are never present in the HTML/JS a student can view — everything sensitive is checked and stored server-side in Apps Script.

02Google Sheet setup

  • Sheet tab named exactly passcode — A1 = label, A2 = the actual passcode value; B1 = label, B2 = due date (ISO string, e.g. 2026-07-31T23:59:00).
  • Sheet tab named exactly Responses — created automatically by the script on first submission (columns: Timestamp, StudentID, StudentName, Score, Total, AnswersJSON).
The due date isn't secret — it lives in the passcode sheet (not Script Properties) purely so it's easy to update without opening the script editor. What matters is that it's checked server-side, so a student can't spoof their local clock to unlock a closed quiz.

03Script Properties

Set under Apps Script → Project Settings → Script Properties. Never exposed to the client.

PropertyExample value
ANSWER_KEY{"q1":"C","q2":"B","q3":"C","q4":"third","q5":"rejecting"}

DUE_DATE is no longer here — it moved to cell B2 of the passcode sheet, since it isn't sensitive and is more convenient to edit there.

How to add a Script Property:

  1. Open the Apps Script editor for the project.
  2. Click the gear icon ⚙ (Project Settings) in the left sidebar.
  3. Scroll to the Script Properties section.
  4. Click Add script property.
  5. Enter the Property name exactly (e.g. ANSWER_KEY) and its Value.
  6. Click Save script properties at the bottom.
  7. No redeploy needed — Script Properties are read live on the next request.

04Deployment

  1. Apps Script editor → Deploy → New deployment → Web app
  2. Execute as: Me  ·  Who has access: Anyone
  3. Authorize access, copy the /exec URL
  4. Paste that URL into the SCRIPT_URL constant in quiz.html
Future edits to Code.gs require redeploying via Deploy → Manage deployments → edit icon → Version: New version → Deploy. This keeps the same /exec URL, so quiz.html doesn't need updating each time.

06Security notes

  • The passcode gate in quiz.html is a UX convenience, not real security — anyone editing the page's JS in dev tools could bypass it. The actual enforcement happens server-side in doPost.
  • Grading logic and the answer key live only in Script Properties, read only inside Apps Script.
  • Due date is checked against the server clock, not the student's device clock.
  • Duplicate submissions are blocked by matching on Student ID.
  • The Google Sheet itself stays private — the script runs "as Me," so students never need direct Sheet access.

07Testing tips

  • Don't test doGet/doPost by clicking Run inside the Apps Script editor — they require a real request object. Test via the deployed URL or the actual quiz.html page.
  • To preview the dashboard before the real due date, temporarily set cell B2 in the passcode sheet to a future value, submit a test response, then set it to a past value to view the dashboard — then set it to the real due date when done. No redeploy needed since it's read directly from the sheet.

08Optional: emailed dashboard

The generateDashboard() function (separate from the on-page dashboard) can be scheduled via Triggers → Add Trigger → time-driven, to email a summary automatically after the due date.
